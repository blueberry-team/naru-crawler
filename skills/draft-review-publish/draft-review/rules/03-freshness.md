# 채용 현행성 검증

02-url-validation 에서 원본 페이지에 접근한 상태에서 수행한다.

## 1. 마감/종료 키워드 스캔

원본 페이지 본문에서 아래 키워드를 검색한다.

**일본어:**
- `募集終了`
- `受付終了`
- `締め切り`
- `締切`
- `終了しました`
- `クローズ`
- `掲載終了`
- `現在募集しておりません`
- `募集は終了`

**영어:**
- `closed`
- `no longer accepting`
- `position has been filled`
- `expired`

**한국어 (나루 측 페이지인 경우):**
- `마감`
- `종료`
- `채용 완료`

키워드가 발견되면:
- HOLD — "원본 페이지에서 채용 종료 키워드 발견: {키워드}"

## 2. deadline 비교

DB의 `deadline` 필드와 오늘 날짜를 비교한다.

| 상태 | 처리 |
|------|------|
| deadline 이 오늘 이후 | PASS |
| deadline 이 오늘 | PASS (당일까지 유효) |
| deadline 이 오늘 이전 | 원본 페이지 재확인 → 여전히 모집 중이면 deadline PUT 갱신, 종료 확인되면 HOLD |
| deadline 이 null | PASS (수시 채용으로 간주, 키워드 스캔 결과에 의존) |

## 3. 원본 목록 대조 (선택)

시간이 허용되면 해당 기업 채용 페이지의 공고 목록을 Playwright MCP 로 확인하여, 리뷰 대상 공고가 여전히 목록에 존재하는지 대조한다.

목록에서 사라졌다면:
- HOLD — "원본 채용 페이지 목록에서 해당 공고가 제거되었습니다"
