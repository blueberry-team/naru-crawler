# 태그 / searchKeywords / description 규칙

## tags

- 최대 10개
- 일반적인 분야나 특징을 나타내는 단어만 사용
- 서비스명 / 제품명 / 브랜드명 금지
- 구직자가 흥미를 가질 만한 일반어만 사용

## searchKeywords

목적:

- 기업명 자체의 검색 변형만 저장한다.
- 서비스명 / 제품명 / 브랜드명은 절대 넣지 않는다.

필수 후보:

1. 일본어 정식 명칭
2. 법인 포함 명칭
3. 영어 정식 명칭
4. 영어명 공백 변형
5. 약칭
6. 읽는 법(히라가나/가타카나)
7. 한국어 표기 + 한국어 음차 흔들림

금지:

- 서비스명 / 제품명 / 브랜드명
- 일반명사
- 대소문자만 다른 변형

허용:

- 영어 공백 변형만 허용
  - 예: `CyberAgent` / `Cyber Agent`

한국어 음차 흔들림:

- `つ / ツ / ッ` 계열이 포함되면 `쯔` / `쓰` / `츠` 변형을 모두 생성한다.
- `ジ/ヂ` → `지`
- `チ` → `치`
- `ン` → `ㄴ` / `은`
- 장음 유무
- 공백 유무
- `サ/ザ` 계열은 자연스러운 범위에서 변형

공통 규칙:

- 중복 제거
- 최대 30개
- 하나의 배열에 일본어, 영어, 한국어 변형을 함께 넣는다
- 번역으로 새 키워드를 임의 생성하지 않는다
- 필수 음차 흔들림 규칙에 해당하는 변형은 빠짐없이 모두 생성한다
- 영어명은 `기본형 1개 + 공백 변형` 만 허용
- 예: `CyberAgent`, `Cyber Agent`
- 금지: `cyberagent`, `CYBERAGENT`

## description

목적:

- 사용자가 회사상을 바로 떠올릴 수 있는 소개문

규칙:

- 1~4행
- 대략 120~280자
- 불릿 금지
- 추측 금지

우선 소스:

1. 공식 사이트의 사업내용 / 서비스 / 회사개요 / 제품·솔루션 설명
2. 채용 사이트의 사업 소개 / 부문 소개 / 업무 소개
3. IR의 사업 세그먼트 설명

추출 순서 권장:

1. 어떤 회사인지
2. 무엇을 주로 하는지
3. 구조적 강점 / 특징
4. 대표 브랜드 / 서비스는 공식상 `주력`, `대표`, `주요` 로 확인될 때만 자연스럽게 1~2개 삽입

브랜드 / 서비스명을 넣을 때 규칙:

- 괄호로 문장 끝에 몰아넣지 않는다.
- 문장 안에 자연스럽게 녹인다.
  - `〜を運営する`
  - `〜を提供する`
  - `〜ブランドを展開する`

구성 권장:

1. 어떤 회사인지
2. 주요 사업 2~4개
3. 구조적 강점 / 특징
4. 대표 브랜드나 서비스는 공식상 주요하다고 확인될 때만 1~2개 자연스럽게 삽입

금지:

- 과장
- 추측
- 작업 기록처럼 나열하는 문체
- 문장 끝 괄호로 서비스명 몰아넣기
