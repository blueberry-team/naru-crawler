# 내비게이션 규칙

## 필수 수집 방식

- 기본 방식: DOM 텍스트에서 제목, 본문, 표, 주석, FAQ, 푸터의 작은 글씨까지 읽는다.
- 보강 방식: 이미지 텍스트, 배너, 표, 플로우 차트는 스크린샷 + Vision으로 읽는다.

## 1단계: 중도채용 섹션으로 이동

아래 순서로 찾는다.

1. 페이지 내부 링크
2. header
3. footer
4. mega menu

우선 탐색 문자열:

- `中途採用`
- `キャリア採用`
- `経験者採用`
- `中途`
- `キャリア`
- `経験者`
- `募集職種`
- `募集要項`
- `求人一覧`
- `職種一覧`
- `採用情報`
- `採用`
- `求人`
- `応募する`

영문 혼합 사이트:

- `Career`
- `Mid-career`
- `Experienced`
- `Experienced Hire`
- `Professional`
- `Recruiting`
- `Careers`

## ATS / 외부 지원 페이지

- 외부 ATS나 다른 도메인으로 이동해도 계속 진행한다.
- 이동한 페이지에서도 다시 아래 문자열을 찾는다.
  - `中途`
  - `キャリア`
  - `経験者`
  - `募集要項`
  - `職種一覧`
  - `選考フロー`
- 외부 페이지가 지원 폼만 있고 공고 상세가 없으면:
  - 원래 사이트로 돌아간다
  - 실제 상세 정보가 있는 페이지에서 수집한다

## 2단계: 목록에서 상세로 이동

- 각 트랙은 반드시 상세 페이지를 연다.
- 목록 페이지에서 작업을 끝내면 안 된다.
- 하나의 트랙은 하나의 payload가 된다.

## 3단계: 이미지와 구조화 정보 확인

반드시 확인할 것:

- 선발 플로우 이미지
- 근무 형태 이미지
- 급여, 근무지, 조건 표 또는 도식
- 필수 요건 / 우대 요건 표
- 고용 형태 / 레벨 / 직급 표기
- `img` 의 alt
- 작은 주석과 표의 각주

텍스트가 이미지 안에 있으면, Vision으로 읽기 전에는 누락으로 판단하지 않는다.
