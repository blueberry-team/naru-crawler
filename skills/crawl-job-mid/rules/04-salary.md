# salaryMin / salaryMax 규칙

## 핵심 원칙

- 급여는 연봉 JPY 기준으로만 저장한다.
- 타입은 `int` 또는 `null`
- 숫자를 훼손하지 않는다.
- 추측으로 금액을 올리거나 내리지 않는다.
- 명시된 숫자 근거만 사용한다.

## 최우선 규칙

페이지에 아래와 같은 연봉 수치가 명시돼 있으면:

- `年収`
- `年俸`
- `想定年収`
- `予定年収`
- `見込み年収`
- `モデル年収`

그 연봉 수치를 그대로 사용한다.

수당 포함 / 제외 여부로 재조정하지 않는다.

## 우선순위

1. 명시적 `年収` / `年俸`
   - 그대로 사용
   - 계산하지 않음
2. 명시적 `想定年収` / `予定年収` / `見込み年収` / `モデル年収`
   - range 그대로 `salaryMin` / `salaryMax` 에 사용
   - 계산하지 않음
3. 월급만 명시된 경우
   - 월급에서 연봉 환산
   - 상여 개월 수가 명시된 경우에만 더함
4. 근거가 애매함
   - `salaryMin = null`
   - `salaryMax = null`

## 월급 환산 규칙

월급 내역이 분해돼 있으면, 급여 성격으로 명시된 금액만 합산한다.

합산 가능한 항목:

- 基本給（月額）
- 固定残業代 / みなし残業代（月額）
- 残業手当（月額）
  - 고정 금액이 명시된 경우만
- 職務手当（月額）
  - 급여성 수당으로 명확할 때만

salary 에 포함하지 않는 것:

- 住宅手当
- 家賃補助
- 寮社宅
- 引越補助
- 交通費
- 支援金
- ポイント
- 복리후생성 항목 전반
- `別途支給`, `支給あり` 이지만 금액이 없음
- 변동 인센티브
- 고정 금액이 아닌 잔업수당 `別途支給`

## 고정잔업수당 처리

내역이 명시된 경우:

- `月給30万円（基本給24万円＋固定残業代6万円／40時間分）`
- 월급 = `24만 + 6만`

고정잔업수당 포함이라고만 적힌 경우:

- `月給50万円（固定残業代含む）`
- `50万円` 그대로 사용
- 추가 합산하지 않음

## 상여 처리

상여는 총 개월 수가 명시된 경우에만 더한다.

- `賞与◯ヶ月分`
- `合計◯ヶ月`
- 연봉 = `월급 * (12 + 총 상여 개월 수)`

아래처럼 적혀 있어도 개월 수 / 금액이 없으면:

- `賞与あり`
- `賞与あり（年2回）`

추측해서 더하지 않는다.

## salaryMin / salaryMax 규칙

- `salaryMin` = 명시 또는 계산으로 확정된 하한
- `salaryMax` = 상한이 명시되거나 계산 가능한 경우에만 설정

예시:

- `500万〜800万`
  - `salaryMin = 5000000`
  - `salaryMax = 8000000`
- `年俸600万円`
  - `salaryMin = 6000000`
  - `salaryMax = 6000000`
- 월급 환산은 가능하지만 상한이 불명확
  - `salaryMin = 월급 * 12 + 명시된 고정 상여`
  - `salaryMax = null`
- `賞与あり（年2回）` 만 있고 개월 수 / 금액이 없음
  - `salaryMax = null`
  - `salaryMin` 을 `salaryMax` 에 복사하지 않음

## 숫자 파싱 규칙

- `万`, `万円` → `*10000`
- `千円` → `*1000`
- `円` → 그대로 사용
- `〜`, `-` → range 양쪽을 파싱
- 숫자 근거가 애매하면 `null`

## notes 처리

salary 에 넣을 수는 없지만 설명상 중요한 급여 보충 문구는 `targetCandidate.notes` 에 남긴다.
