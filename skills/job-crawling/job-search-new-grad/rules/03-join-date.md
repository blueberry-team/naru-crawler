# joinDate 규칙

`joinDate` 는 필수이며 null 이 될 수 없다.

## 목적

- 입사 예정 시기를 저장한다.
- 추측하지 않는다.
- 아래의 졸업 시기 → 입사 월 규칙은 명시적 결정 규칙으로 적용한다.

## 저장 우선순위

1. 입사 월이 명시돼 있으면 그것을 저장한다.
2. 입사 월이 없고 졸업 시점이 명시돼 있으면 아래 규칙을 적용한다.
3. 둘 다 아니면 `미정`

## 입사 월이 명시된 경우

다음과 같은 한국어 형식으로 정규화한다. 일본어 원문(`年`, `月`)을 그대로 두면 안 된다.

- `{YYYY년 M월}`
- `{YYYY년 M월 이후}`
- `{YYYY년 M월 예정}`
- `{YYYY년 M월경}`
- `{YYYY년 M월 / YYYY년 M월}`

예시:

- `2027/4` → `2027년 4월`
- `2027年10月（予定）` → `2027년 10월 예정`
- `2026年10月入社・2027年4月入社` → `2026년 10월 / 2027년 4월`

## 졸업 시기 기반 규칙

규칙 A:

- `YYYY年3月までに卒業`
- `YYYY年3月卒業予定`
- `YYYY年3月修了予定`
- → `YYYY년 4월`

규칙 B:

- `YYYY年9月までに卒業`
- `YYYY年9月卒業予定`
- `YYYY年9月修了予定`
- → `YYYY년 10월`

규칙 C:

- `YYYY卒` 또는 `YYYY年度卒` 만 명시돼 있고
- 페이지가 명확한 신입 채용 / 본전형 페이지인 경우
- → `YYYY년 4월`

규칙 C 예외:

- `通年`, `応相談`, `別途案内` 도 함께 있으면
- → `미정`

## 강제 fallback

- 어느 규칙에도 맞지 않으면 `미정`

## 금지 사항

- 신입 공고라는 이유만으로 `4월` 를 추론하지 않는다.
- 대상자 조건 텍스트를 `joinDate` 에 섞지 않는다.
- `入社時期:` / `입사시기:` 같은 라벨을 붙이지 않는다.
- 일본어 `年`, `月`, `予定`, `以降`, `頃` 를 그대로 저장하지 않는다.
