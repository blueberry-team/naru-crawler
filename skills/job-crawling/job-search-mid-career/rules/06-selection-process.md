# selectionProcess 규칙

`selectionProcess` 는 사람이 읽었을 때 이해할 수 있는 선발 흐름 설명이다.

## 핵심 규칙

- 명시된 선발 프로세스 정보만 저장한다.
- DOM 텍스트와 이미지 둘 다 본다.
- 어디에도 근거가 없을 때만 `null`

## 탐색 위치

- `募集要項`
- `選考フロー`
- `選考プロセス`
- `選考ステップ`
- `採用の流れ`
- `FAQ`
- `よくある質問`
- `エントリー後の流れ`
- 주석과 작은 글씨

추가로 반드시 확인:

- 플로우 차트 이미지
- 화살표
- 번호
- `STEP1/2/3`
- 그림 주석
- 이미지 안 기한, 횟수, 형식, 면접 담당자 정보

## 이미지 추출 규칙

- 플로우가 온전히 읽히도록 스크린샷을 찍는다.
- Vision으로 이미지 안 텍스트를 읽는다.
- 누락되면 확대 / 분할 스크린샷으로 다시 읽는다.
- 명시된 분기는 유지하되, 새로운 분기를 상상해서 만들지 않는다.

## 출력 스타일

권장 형식:

- `エントリー → 書類選考 → 適性検査(SPI) → 1次面接(現場/エンジニア, オンライン) → 2次面接(人事) → 最終面接(役員, 対面) → 内定`

불릿만 있는 경우에는 화살표로 이어 붙인다.

## 구체화 규칙

명시돼 있으면 아래 정보를 반영한다.

- 면접 횟수
- 면접 담당자
- 온라인 / 대면 / web 같은 형식
- SPI 같은 시험 종류
- `変更となる場合あり` 같은 주석

예시:

- `1次面接(現場エンジニア)`
- `2次面接(人事)`
- `最終面接(役員, 対面)`
- `適性検査(SPI)`

## 금지 사항

- 일반적인 채용 절차를 끼워 넣지 않는다.
- 이미지가 있는데 무시하고 `null` 로 두지 않는다.
- 순서가 보이지 않는 텍스트 뭉치를 그대로 붙여 넣지 않는다.
