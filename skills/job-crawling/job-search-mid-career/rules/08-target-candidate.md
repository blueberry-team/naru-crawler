# targetCandidate 규칙

## 목적

`targetCandidate` 는 아래 정보를 구조적으로 저장한다.

- 지원 조건
- 우대 조건
- 인재상
- 평가 포인트
- 근무 조건 요약
- 예외와 주의사항

## 필수 구조

```json
{
  "mustHave": [],
  "niceToHave": [],
  "idealCandidate": [],
  "selectionPoints": [],
  "workingConditions": [],
  "notes": []
}
```

## 탐색 범위

반드시 확인:

- `応募資格`
- `募集要項`
- `必須要件`
- `歓迎要件`
- `求める人物像`
- `人物像`
- `マインド`
- `こんな方におすすめ`
- `選考基準`
- `評価ポイント`
- `求める力`
- `重視する点`
- `勤務条件`
- `勤務時間`
- `休日`
- `勤務地`
- `働き方`
- `制度`
- `注意事項`
- `備考`
- `補足`
- `注記（※）`
- `FAQ`
- 이미지, 배너, 차트, 표, 플로우 안 텍스트

## mustHave

포함:

- `応募資格`
- `必須要件`
- `必須条件`
- `求める経験（必須）`
- 필수 학력, 졸업, 학년, 취업 경험 조건
- 필수 국적, 비자, 어학, 자격 요건

## niceToHave

포함:

- `歓迎要件`
- `歓迎条件`
- `あると望ましい`
- `尚可`
- `プラス`
- 명확한 우대 기술, 경험, 자격

## idealCandidate

포함:

- `求める人物像`
- `期待する人物像`
- `マインド`
- `行動指針`
- `向いている人`
- `こんな方におすすめ`
- `こんな人と働きたい`

## selectionPoints

포함:

- `評価ポイント`
- `重視する点`
- `選考で見ている点`
- `選考基準`
- 면접 / 제출물 평가 기준

## workingConditions

포함:

- 근무시간
- 휴일 / 휴가
- 소정 근로시간
- 플렉스
- 재량노동
- 배치 / 근무지 결정 방식
- 고용형태
- 수습기간
- 전근 여부
- 출장 여부

여기에는 복지 나열을 넣지 않는다.

## notes

포함:

- `備考`
- `注意事項`
- `注記（※）`
- `FAQ`
- 예외 규칙과 단서 문구
  - 예: `職種により異なる`
  - 예: `配属により異なる`
- 연수 / 온보딩
- 급여 구성 보충 설명
  - 예: `固定残業代◯時間分含む`

## 공통 포맷 규칙

- 배열 요소 1개 = 짧은 문장 1개 또는 불릿 1개
- 본문 나열은 의미 단위로 나눈다
- 중복 제거
- `unknown` 같은 dummy 문자열 금지
- `原則`, `一部`, `場合により`, `〜など` 같은 제한 표현은 유지
- 섹션이 없으면 `[]`
