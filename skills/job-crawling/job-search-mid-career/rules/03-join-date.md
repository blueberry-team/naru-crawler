# joinDate 규칙

`joinDate` 는 필수이며 null 이 될 수 없다.

## 목적

- 입사 가능 시기 또는 입사 예정 시기를 저장한다.
- 추측하지 않는다.
- 중도채용 특유의 `즉시`, `협의`, `가능한 한 빠르게` 같은 표현을 정규화한다.

## 저장 우선순위

1. 입사 월이 명시돼 있으면 그것을 저장한다.
2. 입사 월이 없고 입사 가능 시기에 대한 텍스트가 있으면 아래 규칙을 적용한다.
3. 둘 다 아니면 `미정`

## 입사 월이 명시된 경우

다음과 같은 한국어 형식으로 정규화한다. 일본어 원문(`年`, `月`)을 그대로 두면 안 된다.

- `{YYYY년 M월}`
- `{YYYY년 M월 이후}`
- `{YYYY년 M월 예정}`
- `{YYYY년 M월경}`
- `{YYYY년 M월 / YYYY년 M월}`

예시:

- `2026/7` → `2026년 7월`
- `2026年10月（予定）` → `2026년 10월 예정`
- `2026年7月入社・2026年10月入社` → `2026년 7월 / 2026년 10월`

## 중도채용 표현 규칙

- `入社時期応相談`
- `入社日応相談`
- `入社日は相談の上決定`
- → `입사 시기 협의`

- `できるだけ早く`
- `可能な限り早く`
- `早期入社歓迎`
- → `가능한 한 빠르게`

- `内定後できるだけ早く`
- `決定後できるだけ早く`
- → `내정 후 가능한 한 빠르게`

- `即日`
- `即入社可`
- → `즉시 입사`

- `毎月1日入社`
- `月初入社`
- → `매월 1일 입사`

- `未定`
- `別途案内`
- → `미정`

## 강제 fallback

- 어느 규칙에도 맞지 않으면 `미정`

## 금지 사항

- 대상자 조건, 경력 연수, 고용 형태 텍스트를 `joinDate` 에 섞지 않는다.
- `入社時期:` / `입사시기:` 같은 라벨을 붙이지 않는다.
- 일본어 `年`, `月`, `予定`, `以降`, `頃` 를 그대로 저장하지 않는다.
