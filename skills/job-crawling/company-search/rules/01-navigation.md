# 탐색 규칙

## 기본 원칙

- Playwright MCP로 실제 사이트를 탐색한다.
- 회사 공식 사이트를 우선으로 본다.
- 기업 공식 사이트와 채용 사이트를 모두 확인할 수 있다.
- 텍스트가 아니라 이미지로 된 정보는 screenshot + Vision으로 읽는다.
- 외부 OCR 라이브러리를 설치하지 않는다.
- `pip install` 등으로 OCR 도구 추가를 시도하지 않는다.
- 공식 사이트를 가장 우선하는 기준으로 본다.

- `会社概要`
- `企業情報`
- `Company`
- `About`

- `採用情報`

## 탐색 순서

1. 회사 개요 / 기업 정보
2. 채용 정보
3. 모집요강 / 복리후생
4. 기업 이념
5. IR

## OCR / 이미지 규칙

- 기업 이념, 회사 개요 등 중요 정보가 이미지(`<img>`)로만 있는 경우:
  - 외부 도구 설치를 시도하지 않는다.
  - 해당 영역을 screenshot 으로 캡처한다.
  - Vision으로 텍스트를 읽는다.
- `텍스트가 없어서 수집 불가` 로 끝내지 않는다.

## 외부 소스

- EDINET은 공식 사이트에 정확한 수치가 없을 때만 본다.
- OpenWork는 평점 수집 또는 공식 정보 완전 부재일 때만 제한적으로 쓴다.
- OpenWork 결과는 `companySourceUrls` 에 넣지 않는다.
- 공식 사이트에 근거가 있으면 외부 소스로 덮어쓰지 않는다.
