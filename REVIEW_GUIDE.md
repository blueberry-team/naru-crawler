# 나루 공고 품질 리뷰 가이드

DRAFT 공고를 샘플링하여 다각도로 품질을 점검한다.

---

## 1. URL 유효성 검증

### 1-1. jobSourceUrl 접근 가능 여부
- `jobSourceUrl`에 HTTP GET 요청을 보내 **200 응답**을 받는지 확인
- 3xx 리다이렉트인 경우: 최종 도착 URL이 해당 공고 페이지인지 확인 (채용 메인으로 리다이렉트되면 공고 삭제/만료 의심)
- 4xx/5xx 또는 타임아웃: **즉시 보고** — 죽은 링크

### 1-2. 공고 페이지 내용 일치
- 원본 페이지의 제목과 나루에 적재된 `title`이 의미적으로 일치하는지 확인
- 전혀 다른 공고가 표시되면 매핑 오류 가능성

### 1-3. 나루 공고 상태
- 나루 플랫폼에서 해당 공고가 정상 조회되는지 확인 (API 통해)
- DRAFT 상태가 유지되고 있는지, 의도치 않게 PUBLISHED로 전환되지 않았는지

---

## 2. 채용 현행성 검증 (아직 채용 중인가?)

### 2-1. 원본 사이트 확인
- `jobSourceUrl` 페이지에 **마감**, **종료**, **募集終了**, **受付終了**, **クローズ**, **closed** 등의 키워드가 있는지 스캔
- 페이지가 404이거나 "이 공고는 존재하지 않습니다" 류 메시지 → 채용 종료

### 2-2. 마감일 검증
- `deadline` 필드가 있는 경우, 현재 날짜와 비교하여 이미 지난 공고인지 확인
- 마감일이 없는 수시 채용도 원본에서 상태 변경 여부 체크

### 2-3. 공고 목록 대조
- Playwright MCP로 해당 기업 채용 페이지의 현재 공개 공고 목록을 확인하여, 나루 DRAFT의 `jobSourceUrl`이 여전히 유효한지 확인
- 목록에서 빠졌다면 채용 종료 가능성 높음

---

## 3. 데이터 품질 검증

### 3-1. 필수 필드 완성도
| 필드 | 기준 |
|------|------|
| `title` | 5자 이상, 의미 있는 직무명 포함 |
| `overview` | 50자 이상, 실제 직무 설명 포함 (보일러플레이트 아닌지) |
| `tasks` | 1개 이상, 구체적 업무 내용 |
| `position` | 제목/설명과 일치하는 분류인지 |
| `locations` | 원본 근무지와 매핑이 맞는지 |
| `experienceLevel` | 원본 자격 요건과 부합하는지 |

### 3-2. 분류 정확도
- **`position`**: 예) "프론트엔드 개발자" 공고가 `BACKEND`로 분류되진 않았는지
- **`locations`**: 예) 오사카 근무인데 `TOKYO`로 매핑되진 않았는지
- **`experienceLevel`**: 예) 신졸 공고가 `SENIOR`로 잡히진 않았는지

### 3-3. 텍스트 품질
- `overview`에 HTML 태그 잔여물(`<br>`, `&nbsp;` 등)이 남아 있지 않은지
- 깨진 문자(mojibake)가 없는지
- 일본어가 번역되지 않고 그대로 남아 있지 않은지

---

## 4. 중복 검증

- 동일 `jobSourceUrl`로 이미 적재된 공고가 있는지 확인
- 제목이 거의 동일한 공고가 여러 개 있는지 확인 (URL은 다르지만 내용 중복)

---

## 5. 비즈니스 로직 검증

### 5-1. 기업 정보
- `companySlug`가 나루 DB에 실제 존재하는 기업인지 확인
- 기업명과 공고 출처가 일치하는지

### 5-2. 급여/근무형태
- `salaryMin`/`salaryMax`가 있다면 상식적 범위인지 (예: 연봉 0원, 비현실적 고액 등 이상치)
- `workType`이 원본 설명과 일치하는지 (리모트라고 적혀있는데 `ONSITE` 등)

---

## 6. 리포트 형식

각 리뷰 사이클에서 다음 형식으로 보고:

```
나루 공고 리뷰 (#N회차)
시각: YYYY-MM-DD HH:MM

정상 (N개)
• [ID] 제목 — 원본 URL 정상, 채용 중, 분류 정확

주의 (N개)
• [ID] 제목 — 사유 (예: position 분류 의심)

문제 (N개)
• [ID] 제목 — 사유 (예: 원본 URL 404, 채용 종료 등)

개선 제안
• (있다면 기재)
```

---

## 7. 리뷰 우선순위

샘플링 시 다음 우선순위 적용:
1. **아직 한 번도 리뷰하지 않은 공고** 우선
2. **최근 적재된 공고** (새로 들어온 것부터)
3. **이전 리뷰에서 주의였던 공고** 재검증
4. 이미 정상이었던 공고도 주기적으로 재검증 (URL 만료 등)
